# 医院在线预约挂号系统 — 项目分工与接口映射

> 依据 API.md 与前端代码审计结果，制定 5 人全栈分工方案
> 前端页面可直接修改，硬编码 Mock 数据替换为真实 API 调用

## 一、通用约定

| 项目 | 约定值 |
|------|--------|
| 后端基路径 | `http://localhost:8080/api` |
| 请求格式 | `Content-Type: application/json` |
| Token 传递 | `Authorization: Bearer <token>` |
| 统一响应格式 | `{ code: 200, message: "success", data: {} }` |
| 分页请求参数 | `page`、`pageSize`（默认 1/10） |
| 分页响应格式 | `{ records: [], total: 0, page: 1, pageSize: 10 }` |
| 错误码 | 200 成功 · 400 参数错误 · 401 未登录 · 403 无权限 · 404 不存在 · 500 服务端异常 |
| 字段命名 | 数据库 snake_case → API 响应 camelCase，前端统一 camelCase |
| 订单号生成 | 规则由实现方决定（建议时间戳 + 随机数，如 `2026071712345678`） |

### 2.3 基础设施搭建

- 项目骨架（推荐 Spring Boot 2.7+ / Node.js Express / Go Gin）
- 数据库连接池 + MyBatis-Plus / Sequelize / GORM
- 跨域 CORS 配置
- Axios 封装：
  - 请求拦截器：从 localStorage 取出 Token，注入 `Authorization: Bearer xxx`
  - 响应拦截器：业务 `code !== 200` 弹提示；HTTP 401 时清除 Token 并跳转 `login.html?redirect=当前URL`
  - 所有 API 接口文件统一放置在 `api/` 目录下
- 页面级鉴权守卫：原型是多页应用，不使用前端路由守卫；对受限页面统一在公共鉴权脚本中校验登录态
- Token 存储：`localStorage.setItem('token', xxx)`
- 用户信息存储：`localStorage.setItem('userInfo', JSON.stringify(xxx))`

### 2.4 后端接口

| 方法 | 路径 | 请求参数 | 响应 data | 说明 |
|------|------|----------|-----------|------|
| POST | `/auth/register` | `{ phone, password, confirmPassword, captcha }` | `{ userId }` | 注册 → 返回 userId；`username` 由后端按 `phone` 自动生成/同步 |
| POST | `/auth/login` | `{ phone, password }` | `{ token, userInfo: { id, name, phone, avatar } }` | 登录 → 返回 Token |
| GET | `/auth/me` | Header 携带 Token | `{ userInfo: { id, name, phone, avatar } }` | 验证当前 Token 并返回用户信息 |
| POST | `/auth/change-password` | `{ oldPassword, newPassword, confirmPassword }` | `null` | 修改密码 → 销毁原 Token，需重新登录 |
| POST | `/auth/logout` | 无（Header Token） | `null` | 服务端清除 Token 记录 |

### 2.5 数据库表
`t_user` · `t_config`

### 2.6 改造要点（来自代码审计）

| 问题 | 文件位置 | 改造方案 |
|------|----------|----------|
| Token用 `config.headers.token` | `auth-interceptor.js:9` | 改为 `config.headers.Authorization = 'Bearer ' + token` |
| 状态码 `30001` 做过期判断 | `auth-interceptor.js:22` | 改为拦截 HTTP 401 |
| Mock 数据替换 | `login.html` / `register.html` / `change-password.html` 内联 `<script>` 块 | 替换为 `api/auth.js` 中的 Axios 调用 |
| 注册页无 username 字段，但 `t_user.username` 为必填唯一键 | `register.html` + `t_user` | 后端注册时以 `phone` 自动生成/同步 `username`，前端无需新增输入框 |
