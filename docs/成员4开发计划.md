# 医院在线预约挂号系统 — 成员4开发计划

> **负责模块**：个人中心 + 社交互动  
> **技术栈**：Vue 3 Composition API + Spring Boot + MyBatis-Plus + MySQL  
> **前置依赖**：成员1（认证模块）、成员3（订单模块）

---

## 一、任务概览

成员4 负责两大板块共 **8 个前端页面**、**6 个后端 Controller**、**14 个 API 接口**：

| 板块 | 页面 | 核心功能 | 涉及数据表 |
|------|------|----------|------------|
| 个人中心 | 个人信息 | 查看/编辑资料、头像上传 | 	_user |
| 个人中心 | 就诊人管理 | 增删改查家庭成员 | 	_family_member |
| 个人中心 | 我的挂号订单 | 订单列表、状态筛选、取消订单 | 	_appointment |
| 个人中心 | 我的咨询订单 | 咨询列表、状态筛选 | 	_consult |
| 社交互动 | 我的评价 | 评价列表、提交评价 | 	_review |
| 社交互动 | 我的关注 | 关注列表（医院/医生/疾病 Tab） | 	_follow |
| 社交互动 | 消息通知 | 消息列表、标记已读 | 	_message |
| 社交互动 | 意见反馈 | 提交反馈、反馈列表 | 	_feedback |

### 3.2 各页面详细设计

#### 个人信息页 (ProfileView.vue)
- **查看模式**：头像、用户名、手机号、邮箱、真实姓名、性别、生日
- **编辑模式**：可编辑邮箱、真实姓名、性别、生日；不可编辑用户名、手机号
- **头像上传**：文件选择 → 预览 → 上传 POST /user/avatar → 刷新
- **修改密码入口**：跳转 ChangePasswordView.vue（成员1 已实现）

#### 就诊人管理 (FamilyMemberView.vue)
- 卡片列表展示（姓名、关系、手机号、身份证号）
- 弹窗新增/编辑就诊人
- 确认弹窗删除就诊人
- 预约时可选择已有就诊人

#### 我的挂号订单 (MyAppointmentView.vue)
- 状态 Tab：全部 / 待支付 / 已支付 / 已完成 / 已取消
- 订单卡片：医院名称、医生姓名、就诊人、预约时间、金额、状态
- 操作：待支付→去支付、已支付→取消、已完成→评价

#### 我的咨询订单 (MyConsultView.vue)
- 状态 Tab：全部 / 待支付 / 已支付 / 咨询中 / 已完成 / 已取消
- 订单展示：医生姓名、咨询人、预约时间、时长、金额、状态
- 操作：待支付→去支付、已完成→评价

#### 我的评价 (MyReviewView.vue)
- 评价列表：评价对象、订单类型、星级、内容、时间
- 提交评价弹窗：星级评分（复用 RateStar.vue）+ 文字内容

#### 我的关注 (FollowView.vue)
- Tab 切换：关注的医院 / 关注的医生 / 关注的疾病
- 每个 Tab 内分页列表，点击跳转详情页
- 取消关注按钮

#### 消息通知 (MessageView.vue)
- 消息列表：标题、内容摘要、时间、已读/未读状态
- 点击标记已读

#### 意见反馈 (FeedbackView.vue)
- 提交反馈：类型（功能建议/问题反馈/其他）+ 内容
- 我的反馈列表

## 四、前端 API 层

需新建 **8 个 API 模块文件**（其中 2 个调用成员3 接口）：

| 文件 | 对应 Controller | 接口数 |
|------|----------------|--------|
| pi/user.js | UserController | 3 |
| pi/familyMember.js | FamilyMemberController | 4 |
| pi/appointment.js | AppointmentController（成员3） | 2 |
| pi/consult.js | ConsultController（成员3） | 2 |
| pi/review.js | ReviewController | 2 |
| pi/follow.js | FollowController | 3 |
| pi/message.js | MessageController | 2 |
| pi/feedback.js | FeedbackController | 2 |

### API 接口明细

#### 用户模块
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /user/profile | 获取当前用户信息 |
| PUT | /user/profile | 修改个人信息 |
| POST | /user/avatar | 上传头像（multipart/form-data） |

#### 就诊人模块
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /family-members | 就诊人列表 |
| POST | /family-members | 新增就诊人 |
| PUT | /family-members/{id} | 编辑就诊人 |
| DELETE | /family-members/{id} | 删除就诊人 |

#### 评价模块
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /reviews/my | 我的评价列表（分页） |
| POST | /reviews | 提交评价 { orderType, orderId, doctorId, rating, content } |

#### 关注模块
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /follow/my?type=1/2/3 | 关注列表（1医院/2医生/3疾病） |
| POST | /follow/{type}/{id} | 关注 |
| DELETE | /follow/{type}/{id} | 取消关注 |

#### 消息模块
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /messages | 消息列表（分页） |
| POST | /messages/{id}/read | 标记已读 |

#### 反馈模块
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /feedbacks | 提交反馈 { type, content } |
| GET | /feedbacks/my | 反馈列表（分页） |

#### 订单模块（调用成员3）
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /appointments/my | 挂号订单列表 |
| POST | /appointments/{orderNo}/cancel | 取消挂号 |
| GET | /consults/my | 咨询订单列表 |
| POST | /consults/{orderNo}/cancel | 取消咨询 |
